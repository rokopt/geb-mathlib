> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the two code-level translations between IR codes and SlicePFunctors (Lemma 1 and Lemma 2 / Definition 5 of [HancockMcBrideGhaniMalatestaAltenkirch2013]).

**Architecture:** A new file `IndRec/Slice.lean` provides `IR.sliceCode` (non-recursive, SlicePFunctor→IR) and `IR.toSlicePFunctor` (recursive via `IR.elimAlg`, IR→SlicePFunctor). The three per-case components of the algebra are factored into named functions (`toSlicePFunctorIota`/`Sigma`/`Delta`), following the `interpObjIota`/`Sigma`/`Delta` pattern in `Basic.lean`. A test file mirrors the source following the Container.lean test pattern.

**Tech Stack:** Lean 4, mathlib, the project's `Geb.Mathlib.*` infrastructure.

## Global Constraints

- Constructive-only: no `noncomputable`, minimise `Classical` (per `docs/rules/lean-coding.md` § Constructive-only).
- Recursion through recursors only: use `IR.elimAlg`/`IR.Alg`, never self-recursive `def` (per `docs/rules/lean-coding.md` § Recursion and induction through recursors).
- `set_option linter.checkUnivs false in` before declarations with separated arity universes (per `Basic.lean` pattern).
- `@[expose] public section` wraps the whole file (per `Container.lean` pattern).
- Module docstring with all required sections (per `docs/rules/lean-coding.md` § Documentation).
- Every `def` and `theorem` has a `/-- … -/` docstring.
- Terminology: "shapes" and "directions" (never "positions").
- Commit messages: `feat:` / `test:` / `chore:` prefix, imperative present tense, no capital, no period.
- Universe stabilization: shapes at `max uA uB uI`, directions at `uB`.
- Constraint embedding: `ULift.{uA} (PLift (Prop-valued))` for `Type uA`.
- `match` arms use `=>` (not `↦`).

```lean
# Codes for slice polynomial functors as IR codes, and back

Section 4 of [HancockMcBrideGhaniMalatestaAltenkirch2013] establishes that
small inductive-recursive definitions (IR codes) and dependent polynomials
(indexed containers) define the same class of functors. This file implements
the two code-level translations: `IR.sliceCode` translates a
`SlicePFunctor` to an `IR` code (Lemma 1), and `IR.toSlicePFunctor`
translates an `IR` code to a `SlicePFunctor` (Lemma 2 / Definition 5),
the latter via `IR.elimAlg`.

## Main definitions

* `IR.sliceCode` — the `IR` code for a slice polynomial functor (Lemma 1
  of [HancockMcBrideGhaniMalatestaAltenkirch2013]).
* `IR.toSlicePFunctorIota`, `IR.toSlicePFunctorSigma`,
  `IR.toSlicePFunctorDelta` — the per-case components of the recursive
  translation (Definition 5).
* `IR.toSlicePFunctorAlg` — the `Alg` assembling the three components.
* `IR.toSlicePFunctor` — the slice polynomial functor for an `IR` code
  (Lemma 2 / Definition 5), generated by `IR.elimAlg`.

## Main statements

* `IR.toSlicePFunctor_iota`, `IR.toSlicePFunctor_sigma`,
  `IR.toSlicePFunctor_delta` — the computation rules of
  `IR.toSlicePFunctor` at each code constructor, via `IR.elim_mk`.

## Implementation notes

The recursive translation stabilizes the universe parameters: shapes at
`max uA uB uI` (the delta case introduces shapes indexed by `B → I` at
`Type (max uB uI)`), directions at `uB`. The constraint in
`IR.sliceCode` embeds the `Prop`-valued compatibility condition into
`Type uA` via `ULift.{uA} (PLift (...))`, following the pattern of
`IR.InnerHom` (`IndRec/Hom.lean`). The delta-case direction structure
follows the indexed-container presentation (proof of Lemma 2) rather than
the dependent-polynomial diagram (Definition 5); the two are equivalent
via the fiber construction of Definition 2.

* [HancockMcBrideGhaniMalatestaAltenkirch2013]

@[expose] public section

universe uA uB uI uO

namespace IndRec

open CategoryTheory

variable (I : Type uI) (O : Type uO)

namespace IR

set_option linter.checkUnivs false in
/-- The `IR` code for a slice polynomial functor `F` (Lemma 1 of
[HancockMcBrideGhaniMalatestaAltenkirch2013]): a `sigma` over the
shapes, then for each shape a `delta` over its directions, then a
`sigma` over the compatibility constraint, terminating in `iota` at the
output index. -/
def sliceCode (F : SlicePFunctor.{uA, uB, uI, uO} I O) :
    IR.{uA, uB, uI, uO} I O :=
  sigma I O F.toPFunctor.A fun a =>
    delta I O (F.toPFunctor.B a) fun assign =>
      sigma I O (ULift.{uA} (PLift (∀ b, assign b = F.rCurried a b))) fun _ =>
        iota I O (F.q a)

end IR

end IndRec
```

```lean
/-- A test slice polynomial functor: shape `Bool`, directions `Nat`,
all directions map to `PUnit.unit`, all shapes map to `PUnit.unit`. -/
def testSlice : SlicePFunctor.{0, 0, 0, 0} PUnit PUnit :=
  { toPFunctor := ⟨Bool, fun _ ↦ Nat⟩
  , r := fun _ ↦ PUnit.unit
  , q := fun _ ↦ PUnit.unit }

/-- The `IR` code for `testSlice`. -/
def testSliceCode : IR.{0, 0, 0, 0} PUnit PUnit :=
  IR.sliceCode PUnit PUnit testSlice

`delta` over directions, `sigma` over the compatibility constraint,
`iota` at the output index. -/

        IR.delta PUnit PUnit Nat fun assign =>
          IR.sigma PUnit PUnit
            (ULift.{0} (PLift (∀ b, assign b = testSlice.rCurried a b))) fun _ =>
            IR.iota PUnit PUnit PUnit.unit :=
  rfl
```

### Task 2: Add `IR.toSlicePFunctorAlg`, `IR.toSlicePFunctor`, and computation rules

**Files:**
- Modify: `Geb/Mathlib/Data/PFunctor/IndRec/Slice.lean` (add before `end IR`)

**Interfaces:**
- Consumes: `IR.elimAlg`, `IR.Alg`, `IR.elim_mk` from `Basic.lean`; `IR.sliceCode` from Task 1.
- Produces: `IR.toSlicePFunctorIota`, `IR.toSlicePFunctorSigma`, `IR.toSlicePFunctorDelta`, `IR.toSlicePFunctorAlg`, `IR.toSlicePFunctor`, `IR.toSlicePFunctor_iota`, `IR.toSlicePFunctor_sigma`, `IR.toSlicePFunctor_delta`.

**Background — the `Alg` interface:**

`IR.Alg I O V` is a triple (a `Dest I O V V`) with three components:
- `.1 : O → V` — iota: `(o : O) → SlicePFunctor I O`
- `.2.1 : (A : Type uA) → (A → V) → V` — sigma: takes sub-results `A → V`
- `.2.2 : (B : Type uB) → ((B → I) → V) → V` — delta: takes sub-results `(B → I) → V`

The sub-results in the sigma/delta cases are plain functions (no `ULift`
wrapping); `Dest.elim` handles the `ULift.up`/`ULift.down` conversion
internally. `elimAlg I O V alg` produces an `IR I O → V` whose
computation rule `elim_mk` holds by `rfl` (definitional).

- [ ] **Step 1: Add the three per-case functions and the algebra**

Insert the following before `end IR` in `Slice.lean` (after `sliceCode`):

```lean
set_option linter.checkUnivs false in
/-- The constant (`iota`) case of the translation from `IR` codes to
`SlicePFunctor`s (Definition 5, clause 1 of
[HancockMcBrideGhaniMalatestaAltenkirch2013]): one shape, no directions,
output index `o`. -/
def toSlicePFunctorIota (o : O) :
    SlicePFunctor.{max uA uB uI, uB, uI, uO} I O :=
  { toPFunctor := ⟨PUnit, fun _ ↦ PEmpty⟩
  , r := fun ⟨_, b⟩ ↦ PEmpty.elim b
  , q := fun _ ↦ o }

set_option linter.checkUnivs false in
/-- The dependent sum (`sigma`) case of the translation (Definition 5,
clause 2): coproduct of sub-polynomials — shapes, directions, input map,
and output map inherited componentwise. -/
def toSlicePFunctorSigma (A : Type uA)
    (sub : A → SlicePFunctor.{max uA uB uI, uB, uI, uO} I O) :
    SlicePFunctor.{max uA uB uI, uB, uI, uO} I O :=
  { toPFunctor := ⟨Σ a, (sub a).toPFunctor.A,
      fun ⟨a, sa⟩ ↦ (sub a).toPFunctor.B sa⟩
  , r := fun ⟨⟨a, sa⟩, p⟩ ↦ (sub a).r ⟨sa, p⟩
  , q := fun ⟨a, sa⟩ ↦ (sub a).q sa }

set_option linter.checkUnivs false in
/-- The dependent product (`delta`) case of the translation (Definition 5,
clause 3): shapes indexed by assignments `B → I`, directions are the
coproduct of the arity and the sub-directions, the input map is the
cotuple `[i, sub.r]`, the output map delegates. -/
def toSlicePFunctorDelta (B : Type uB)
    (sub : (B → I) → SlicePFunctor.{max uA uB uI, uB, uI, uO} I O) :
    SlicePFunctor.{max uA uB uI, uB, uI, uO} I O :=
  { toPFunctor := ⟨Σ (i : B → I), (sub i).toPFunctor.A,
      fun ⟨i, sa⟩ ↦ Sum B ((sub i).toPFunctor.B sa)⟩
  , r := fun ⟨⟨i, sa⟩, p⟩ ↦ match p with
    | Sum.inl b => i b
    | Sum.inr p' => (sub i).r ⟨sa, p'⟩
  , q := fun ⟨i, sa⟩ ↦ (sub i).q sa }

set_option linter.checkUnivs false in
/-- The algebra computing one step of the translation from `IR` codes to
`SlicePFunctor`s (Definition 5 of
[HancockMcBrideGhaniMalatestaAltenkirch2013]), assembling the iota,
sigma, and delta cases. -/
def toSlicePFunctorAlg :
    Alg.{uA, uB, uI, uO, max (max uA uB uI + 1) (uB + 1) uI uO} I O
      (SlicePFunctor.{max uA uB uI, uB, uI, uO} I O) :=
  ⟨toSlicePFunctorIota I O, toSlicePFunctorSigma I O, toSlicePFunctorDelta I O⟩
```

- [ ] **Step 2: Add `toSlicePFunctor` and computation rules**

Insert the following after `toSlicePFunctorAlg`:

```lean
/-- The slice polynomial functor for an `IR` code (Lemma 2 / Definition 5
of [HancockMcBrideGhaniMalatestaAltenkirch2013]), generated by `IR.elimAlg`
from `IR.toSlicePFunctorAlg`. -/
def toSlicePFunctor (γ : IR.{uA, uB, uI, uO} I O) :
    SlicePFunctor.{max uA uB uI, uB, uI, uO} I O :=
  elimAlg I O (SlicePFunctor.{max uA uB uI, uB, uI, uO} I O)
    (toSlicePFunctorAlg I O) γ

/-- Computation rule for `IR.toSlicePFunctor` at `IR.iota` (definitional,
via `IR.elim_mk`). -/
theorem toSlicePFunctor_iota (o : O) :
    toSlicePFunctor I O (iota I O o) = toSlicePFunctorIota I O o := rfl

/-- Computation rule for `IR.toSlicePFunctor` at `IR.sigma` (definitional,
via `IR.elim_mk`). -/
theorem toSlicePFunctor_sigma (A : Type uA) (c : A → IR I O) :
    toSlicePFunctor I O (sigma I O A c) =
      toSlicePFunctorSigma I O A fun a => toSlicePFunctor I O (c a) := rfl

/-- Computation rule for `IR.toSlicePFunctor` at `IR.delta` (definitional,
via `IR.elim_mk`). -/
theorem toSlicePFunctor_delta (B : Type uB) (c : (B → I) → IR I O) :
    toSlicePFunctor I O (delta I O B c) =
      toSlicePFunctorDelta I O B fun i => toSlicePFunctor I O (c i) := rfl
```
